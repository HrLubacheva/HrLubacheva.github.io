# Документация сайта hrlubacheva.github.io (подробная)

## 1. Общее описание

Проект представляет собой одностраничный сайт-визитку для карьерного консультанта, HR-эксперта и бизнес-тренера **Виктории Любачевой**.

### Основные функции
- **Калькулятор услуг** – выбор услуг из 9 категорий, корзина, скидка 5% при заказе от 2 позиций, кнопка «Скопировать корзину».
- **Карьерный квиз** – 7 вопросов (роль, уровень, срочность, важность, бюджет, отрасль, формат работы). На основе весов услуг подбирает 2 оптимальных варианта.
- **Форма бесплатной 20‑минутной диагностики** – обратный звонок с валидацией телефона и email.
- **Бесплатные материалы** – чек-лист «Идеальное резюме» и программа тренинга «Продай себя дорого» (скачать PDF или получить на email).
- **Карусель сертификатов** – автопрокрутка (5 сек), остановка при наведении, ручное листание стрелками, точками, перетаскиванием мышью, колёсиком мыши.
- **Полный прайс-лист** – два аккордеона (для бизнеса и для соискателей) с группировкой по категориям.
- **Адаптивный дизайн**, плавные анимации при скролле.
- **Отправка заявок в Google Sheets** + уведомления в Telegram и на email администратора (чистые сообщения без мусора).

### Технологии
- **Frontend:** HTML5, CSS3 (переменные, гриды, флексы), JavaScript (ES6)
- **Backend:** Google Apps Script (обработка форм, отправка email/Telegram, rate limiting)
- **Сборка:** Python 3 (скрипт `build.py`)
- **Хостинг:** GitHub Pages
- **Аналитика:** Яндекс.Метрика, Google Analytics (с согласием cookie)

## 3. Настройка бэкенда (Google Apps Script)

### 3.1. Создание проекта

1. Перейдите на [script.google.com](https://script.google.com)
2. Создайте новый проект → **Пустой проект**.
3. Назовите, например: `hrlubacheva-backend`.
4. Удалите весь код из редактора и вставьте код из файла `Code.gs` (см. последнюю версию с чистыми уведомлениями и rate limiting).

### 3.2. Переменные окружения (PropertiesService)

В редакторе Apps Script откройте **Настройки проекта** → **Свойства скрипта** → **Добавить свойство**.

| Ключ | Описание | Где взять |
|------|----------|-----------|
| `SPREADSHEET_ID` | ID Google Таблицы | В URL таблицы между `/d/` и `/edit` |
| `TELEGRAM_BOT_TOKEN` | Токен бота Telegram | Создать бота у [@BotFather](https://t.me/BotFather) |
| `TELEGRAM_CHAT_ID` | ID чата для уведомлений | Написать боту `@userinfobot` |
| `ADMIN_EMAIL` | Email администратора | Ваш email (например, hrlubacheva@yandex.ru) |

### 3.3. Настройка Google Sheets

- Скрипт самостоятельно создаст все необходимые листы при первом запросе:
  - `Обратная связь`
  - `Быстрый заказ`
  - `Материалы`
  - `Квиз ответы`
  - `Технические ошибки`
  - `Логи`
  - `Другое`

### 3.4. Публикация как веб-приложения

1. **Развернуть** → **Новое развертывание**.
2. Тип: **Веб-приложение**.
3. Выполнять от: **Я**.
4. Кто имеет доступ: **Все, даже без аутентификации**.
5. Нажмите **Развернуть** → скопируйте URL (например, `https://script.google.com/macros/s/.../exec`).
6. Вставьте этот URL в `components/common/js/00_config.js` как значение `SCRIPT_URL`.

### 3.5. Проверка работы

- Отправьте тестовую заявку через форму на сайте.
- Проверьте появление данных в таблице, уведомления в Telegram и на email.

## 4. Как добавлять новый контент

### 4.1. Добавление услуги в калькулятор

1. Откройте `components/common/js/09_services-data.js`.
2. Найдите массив нужной категории (например, `start` для раздела «Ищу работу»).
3. Добавьте запись: `{name: "Название услуги", price: 10000}`.
   - `price: null` – цена по запросу, `price: 0` – бесплатно.
4. Запустите `python build.py`.

### 4.2. Изменение весов в квизе

1. Откройте `components/common/js/11_quiz-weights.js`.
2. Найдите или добавьте блок для услуги:
   ```js
   "Название услуги": {
     role_job_seeker: 10,
     level_middle: 7,
     urgency_fast: 9,
     // … остальные ключи
   },
   ```
3. Ключевые веса: `role_*`, `level_*`, `urgency_*`, `importance_*`, `budget_*`, `industry_*`, `work_format_*`.

### 4.3. Замена изображений

| Изображение | Путь | Рекомендуемый размер |
|-------------|------|----------------------|
| Главное фото | `assets/images/01-hero.webp` | 480×640 |
| OG‑изображение | `assets/images/og-image.jpg` | 1200×630 |
| Фото на сцене | `assets/images/03-process-stage.webp` | произвольный |
| Фото тренинга | `assets/images/02-process-training.webp` | произвольный |
| ТВ‑фото | `assets/images/media-tv-*.webp` | 320×180 |
| Сертификаты | `assets/images/certs/*.webp` | ≈260×260 (вертикальные), 360×260 (горизонтальные) |

После замены выполните `python build.py` и залейте изменения в репозиторий.

### 4.4. Обновление PDF-файлов

Замените файлы в `assets/docs/`, затем `python build.py` и git push.

### Без реального бэкенда

- В консоли браузера выполните `enableLogs()`, отправьте форму и следите за выводом.

## 7. Известные ограничения и решения

| Проблема | Решение |
|----------|---------|
| **ipapi.co может блокироваться** | Геоданные не критичны; в таблицу уйдёт «-». |
| **Лимиты Google Apps Script** | На бесплатном аккаунте: 100 email/день, 20 запусков/мин. При высоком трафике – платный тариф. |
| **Telegram не приходят уведомления** | Проверьте токен и chat ID, бот должен быть запущен (напишите ему `/start`). |
| **CORS при локальной разработке** | В `Code.gs` добавлены заголовки `Access-Control-Allow-Origin: *`. Используйте `server.py`. |
| **Анимация .fade-up не срабатывает** | Переопределение `initCalculator` и `initQuiz` в `99_main.js` перезапускает анимацию. |

## 8. Важные технические нюансы

### 8.1. Анимация `fade-up`
- Элементу с классом `.fade-up` изначально заданы `opacity: 0; transform: translateY(30px);`.
- При попадании в область видимости добавляется класс `.visible` (через Intersection Observer).

### 8.2. Кастомные выпадающие списки (калькулятор)
- Обычные `<select class="service-select">` скрываются (`display: none`).
- Взамен создаётся `<div class="custom-dropdown">` с кнопкой и меню.

### 8.3. Карусель сертификатов
- Автопрокрутка (1 слайд / 5 сек), прогресс-бар.
- При наведении мыши автопрокрутка останавливается, можно перетаскивать слайды мышью или листать колёсиком.
- Кнопки «вперёд/назад», точки навигации.

### 8.4. Формы и валидация
- Телефон маскируется и валидируется на 11 цифр (с ведущей 7).
- Поле email проверяется на корректность (необязательно).
- Чекбокс согласия обязателен.

### 8.5. Cookie-баннер и аналитика
- Баннер появляется при первом визите, стилизован под общий дизайн (иконка печеньки, кнопки).
- После нажатия «Принять» активируются Яндекс.Метрика и Google Analytics.

### 8.6. Блок выбранной услуги из квиза
- `#quizSelectionBlock` стилизован как элемент корзины калькулятора.
- На десктопе – сетка, на мобильных – вертикальная карточка с крестиком.
