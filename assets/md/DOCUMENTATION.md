## 3. Описание ключевых JavaScript-модулей

### 00_core.js
- Содержит вспомогательные функции: `escapeHtml`, `log*`, `showToast`, `getOrCreateLocalUserId`, `sendDataToSheet`, `formatPhoneNumber`, `getUTMParams`, `getGeoData` и т.д.
- Определяет `IS_DEV` (отключает логи в продакшене).
- Сохраняет `user_id` в `localStorage` для аналитики.
- Отвечает за сбор геоданных через `ipapi.co`.
- **Валидация телефона и email**: функции `validatePhoneDigits`, `validateEmailFormat`, `showFieldError`, `clearFieldError`, `validatePhoneField`, `validateEmailField`, `bindLiveValidation`. При ошибке поле подсвечивается красным, под ним показывается сообщение.

### 07_calculator.js
- Управляет корзиной (`cart`).
- Функция `initCustomDropdowns` заменяет стандартные `<select>` на кастомные кнопки с выпадающим меню.
- Функция `renderCart` отображает добавленные услуги, считает итог и скидку.
- Данные услуг берутся из `window.LOCAL_SERVICES` (файл `07.services-data.js`).

### 08_quiz.js
- Реализует квиз с 5 вопросами.
- В зависимости от выбранного уровня, второй вопрос меняется (для рекрутеров – другой текст).
- После ответов вызывается `window.getTopTwoServices` (из `08.quiz_weights.js`), которая возвращает две наиболее подходящие услуги.
- Пользователь может выбрать одну из них или запросить помощь.
- Выбранный вариант сохраняется в скрытых полях формы и отображается в специальном блоке `#quizSelectionBlock`, который стилизован так же, как элементы корзины калькулятора.

### 02_forms.js
- Обрабатывает отправку формы обратного звонка и быстрого заказа.
- **Перед отправкой** выполняет валидацию телефона (обязательно 11 цифр) и email (если заполнен). Если поля не проходят – показывает тост и подсвечивает ошибку, форма не отправляется.
- При успешной валидации вызывает `window.submitForm`, который собирает все данные (гео, UTM, время на сайте, корзину и т.д.) и отправляет POST-запросом на `SCRIPT_URL`.

### 99_public-main.js
- Главный скрипт, запускается после загрузки DOM.
- Инициализирует все модули: анимацию, формы, модалки, калькулятор, квиз, навигацию, cookie, кнопку «Наверх».
- Переопределяет `initCalculator` и `initQuiz`, чтобы после их выполнения перезапускать анимацию для новых элементов.
- Вешает обработчик на клик по вкладкам калькулятора – анимация обновляется.
- При `load` принудительно показывает элементы с классом `.fade-up`, которые уже видны.

## 4. Как работать с проектом (пошагово)

### 4.1. Локальная разработка

```bash
# 1. Собрать index.html из компонентов
python build.py

# 2. Запустить локальный сервер (порт 8080)
python server.py
```

Сайт станет доступен по адресу `http://127.0.0.1:8080`.  
При любом изменении в `components/` нужно **заново выполнить `build.py`**, затем обновить страницу в браузере (лучше `Ctrl+Shift+R`, чтобы сбросить кеш).

### 4.2. Редактирование контента

- **Текст, HTML-структура** – править файлы в `components/sections/`.
- **Глобальные стили** – только `components/common/css/00_base.css`.
- **JavaScript-логика** – править соответствующие файлы в `components/common/js/`.

### 4.3. Добавление новой секции

1. Создать новый HTML-файл в `components/sections/` с именем, например, `12_clients.html`.
2. Вставить код секции. Рекомендуется добавить контейнеру класс `fade-up`, если нужна анимация при скролле.
3. Запустить `build.py`. Секция появится в конце страницы (порядок сортируется по имени файла).

### 4.4. Обновление PDF-файлов

1. Заменить `checklist.pdf` и/или `training_program.pdf` в папке `assets/docs/`.
2. Сохранить те же имена.
3. Выполнить `git add`, `commit`, `push`.

Ссылки на сайте остаются теми же, файлы обновятся автоматически.

### 4.5. Публикация на GitHub Pages

```bash
python build.py               # собрать свежий index.html
git add .
git commit -m "описание изменений"
git push origin main
```

Через 1–2 минуты сайт обновится по адресу `https://hrlubacheva.github.io/`.

## 5. Важные технические нюансы

### 5.1. Анимация `fade-up`
- Элемент с классом `.fade-up` изначально прозрачен и сдвинут вниз.
- При попадании в область видимости (через Intersection Observer) добавляется класс `.visible`, который делает его видимым.
- В `99_public-main.js` есть функция `reinitAnimations`, которая следит за новыми элементами, добавляемыми после подгрузки калькулятора или квиза.

### 5.2. Кастомные выпадающие списки
- Обычные `<select class="service-select">` скрываются (`display: none`).
- Взамен создаётся `<div class="custom-dropdown">` с кнопкой и меню.
- Стили описаны в `00_base.css` (блок в конце файла). Поддерживают плавную анимацию, тени, акцент при наведении, скроллбар.

### 5.3. Адаптивные формы (валидация и единый стиль)
- В `00_base.css` добавлены классы `.responsive-form`, `.form-row`, `.btn-block`. Они делают поля и кнопки шириной 100% на мобильных.
- Поля телефона и email проходят **живую валидацию**: при потере фокуса проверяется корректность, при ошибке поле подсвечивается красным, появляется сообщение под ним.
- **Блок выбранной услуги из квиза** (`#quizSelectionBlock`) полностью копирует стили элементов корзины калькулятора:
  - На десктопе – сетка из трёх колонок (название, цена, кнопка удаления).
  - На мобильных – вертикальная карточка (название, цена, крестик в правом верхнем углу).
- При пустом выборе блок скрывается (через JS и CSS-правило `:empty`).

### 5.4. Cookie-баннер и аналитика
- Яндекс.Метрика и Google Analytics загружаются сразу, но по умолчанию имеют `analytics_storage: 'denied'`.
- После нажатия «Принять» в `localStorage` сохраняется `cookie_consent = 'accepted'`, и счётчики обновляют настройки через `gtag('consent', 'update', ...)`.
- Если пользователь отклонил – баннер больше не показывается, аналитика не активируется.

### 5.5. Формы и отправка данных
- Все формы (обратный звонок, быстрый заказ, запрос материалов) собирают данные и отправляют POST-запросом на `SCRIPT_URL` (Google Apps Script).
- Перед отправкой происходит **валидация телефона и email** (на стороне клиента). Если поля не проходят – показывается тост, форма не отправляется.
- В данные автоматически добавляются: `userId`, `utm`, `device`, `page`, `geo`, `timeOnSite`, `visitStats`, а также содержимое корзины и выбранный вариант из квиза.
- В файле `00_config.js` указан `SCRIPT_URL` – ссылка на веб-приложение Google Apps Script. Сам скрипт лежит на стороне Google, его код не входит в репозиторий.

### 5.6. Service Worker (`sw.js`)
- Файл создан, чтобы браузер не выдавал ошибку 404 при попытке зарегистрировать Service Worker.
- Он абсолютно пустой (только заглушка), не кеширует ничего. Сайт и так работает офлайн благодаря стандартному кешированию браузера.
