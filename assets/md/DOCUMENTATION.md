## 3. Настройка бэкенда (Google Apps Script)

### 3.1. Создание проекта в Google Apps Script

1. Перейдите на [script.google.com](https://script.google.com)
2. Создайте новый проект: **Новый проект** → **Пустой проект**
3. Назовите проект, например: `hrlubacheva-backend`
4. Удалите весь код из редактора и вставьте код из файла `Code.gs` (прилагается отдельно)
5. Нажмите **Сохранить** (Ctrl+S)

### 3.2. Настройка переменных окружения (PropertiesService)

В редакторе Apps Script перейдите в **Настройки проекта** (шестерёнка слева) → **Свойства скрипта** → **Добавить свойство**.

Добавьте следующие ключи и значения:

| Ключ | Описание | Где взять |
|------|----------|-----------|
| `SPREADSHEET_ID` | ID Google Таблицы | Открыть таблицу → в URL между `/d/` и `/edit` |
| `TELEGRAM_BOT_TOKEN` | Токен бота Telegram | Создать бота у [@BotFather](https://t.me/BotFather) → получить токен |
| `TELEGRAM_CHAT_ID` | ID чата для уведомлений | Написать боту `@userinfobot` → скопировать свой ID |
| `ADMIN_EMAIL` | Email для уведомлений | Ваш email (например, hrlubacheva@yandex.ru) |

### 3.3. Настройка Google Sheets

1. Создайте новую таблицу в Google Sheets
2. Скопируйте её ID из URL (между `/d/` и `/edit`)
3. Вставьте ID в переменную `SPREADSHEET_ID`
4. **Важно:** Скрипт сам создаст нужные листы (`Обратная связь`, `Быстрый заказ`, `Материалы`, `Квиз ответы`, `Логи`) при первом запуске

### 3.4. Публикация как веб-приложения

1. В редакторе нажмите **Развернуть** → **Новое развертывание**
2. Тип: **Веб-приложение**
3. Выполнять от: **Я**
4. Кто имеет доступ: **Все, даже без аутентификации**
5. Нажмите **Развернуть**
6. Скопируйте полученный URL (он будет выглядеть как `https://script.google.com/macros/s/.../exec`)
7. Вставьте этот URL в `components/common/js/00_config.js` как значение `SCRIPT_URL`

### 3.5. Проверка работы бэкенда

1. Отправьте тестовую заявку через форму на сайте
2. Проверьте:
   - Появились ли данные в Google Sheets
   - Пришло ли уведомление в Telegram
   - Пришло ли уведомление на email администратора

## 4. Как добавлять новый контент (частые сценарии)

### 4.1. Добавление новой услуги в калькулятор

1. Откройте `components/common/js/07.services-data.js`
2. Найдите нужную категорию (например, `start: [` для раздела "Ищу работу")
3. Добавьте новую запись:
   ```javascript
   {name: "Название услуги", price: 10000},
   ```
   - Если цена по запросу: `price: null`
   - Если бесплатно: `price: 0`
4. Запустите `python build.py`
5. Проверьте, что услуга появилась в выпадающем списке

### 4.2. Изменение весов в квизе

1. Откройте `components/common/js/08.quiz_weights.js`
2. Найдите или добавьте запись для услуги:
   ```javascript
   "Название услуги": {
     role_job_seeker: 10,
     role_career_change: 8,
     level_middle: 7,
     urgency_fast: 9,
     // ... остальные веса
   },
   ```
3. **Ключевые веса для понимания:**
   - `role_*` — роль пользователя (ищу работу, рост, бизнес)
   - `level_*` — уровень (junior, middle, senior, lead)
   - `urgency_*` — срочность
   - `importance_*` — приоритеты
   - `budget_*` — бюджет

### 4.3. Добавление вопроса в квиз

1. Откройте `components/common/js/09_quiz.js`
2. Добавьте новый вопрос в массив `quizQuestions` (в функции `startQuiz`)
3. Увеличьте размер массива `answers` с 5 до нужного количества
4. Обновите логику расчёта в `renderResult`

### 4.4. Замена изображений

| Изображение | Путь | Размер |
|-------------|------|--------|
| Главное фото | `assets/images/img_1.jpg` | ~480x640 |
| OG-изображение | `assets/images/og-image.jpg` | 1200x630 |
| Фото на сцене | `assets/images/img_4.jpg` | произвольный |
| Фото тренинга | `assets/images/img_7.jpg` | произвольный |
| Иконки | `assets/images/apple-touch-icon.png` | 180x180 |
| Android-иконка | `assets/images/android-icon-192x192.png` | 192x192 |

**После замены:**
```bash
python build.py
git add assets/images/
git commit -m "update images"
git push
```

### 4.5. Обновление PDF-файлов

1. Замените файлы в `assets/docs/`:
   - `checklist.pdf` — чек-лист "Идеальное резюме"
   - `training_program.pdf` — программа тренинга
   - `docs.zip` — архив с обоими файлами
2. Запустите сборку и залейте на GitHub

## 6. Локальная отладка форм

### Без реального бэкенда

Если вы хотите тестировать отправку форм без подключения к Google Apps Script:

1. Откройте `components/common/js/00_config.js`
2. Временно замените `SCRIPT_URL` на тестовый endpoint:
   ```javascript
   SCRIPT_URL: 'https://httpbin.org/post',  // тестовый echo-сервер
   ```
3. Соберите и запустите сайт
4. Все отправки форм будут уходить на тестовый сервер

### С реальным бэкендом

1. Разверните Google Apps Script (см. раздел 3)
2. Убедитесь, что `SCRIPT_URL` указывает на ваш веб-приложение
3. Включите логи в браузере: в консоли выполните `enableLogs()`
4. Отправьте форму и следите за консолью

## 7. Известные ограничения и решения

| Проблема | Решение |
|----------|---------|
| **ipapi.co может блокироваться** | Данные геолокации не критичны для работы форм. Если сервис недоступен, в таблицу уйдёт значение "-" |
| **Лимиты Google Apps Script** | На бесплатном аккаунте: 100 email/день, 20 запусков/мин. При высоком трафике рекомендуется перейти на платный тариф |
| **Telegram уведомления не приходят** | Проверьте `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`. Бот должен быть запущен (напишите ему `/start`) |
| **CORS при локальной разработке** | В `Code.gs` добавлены заголовки `Access-Control-Allow-Origin: *`. Для локальной разработки используйте `server.py` |
| **Анимация не работает после переключения вкладок** | В `99_public-main.js` есть переопределение `initCalculator` и `initQuiz`, которые перезапускают анимацию |

## 8. Важные технические нюансы

### 8.1. Анимация `fade-up`
- Элемент с классом `.fade-up` изначально прозрачен и сдвинут вниз.
- При попадании в область видимости добавляется класс `.visible`.
- В `99_public-main.js` есть функция `reinitAnimations`, которая следит за новыми элементами.

### 8.2. Кастомные выпадающие списки
- Обычные `<select class="service-select">` скрываются (`display: none`).
- Взамен создаётся `<div class="custom-dropdown">` с кнопкой и меню.
- Поддерживают плавную анимацию, тени, скроллбар.

### 8.3. Адаптивные формы и валидация
- Поля телефона и email проходят живую валидацию: при потере фокуса проверяется корректность.
- При ошибке поле подсвечивается красным, появляется сообщение под ним.
- Форма не отправляется, пока все обязательные поля не пройдут валидацию.

### 8.4. Блок выбранной услуги из квиза
- Блок `#quizSelectionBlock` стилизован так же, как элементы корзины калькулятора.
- На десктопе – сетка из трёх колонок (название, цена, кнопка удаления).
- На мобильных – вертикальная карточка с крестиком в правом верхнем углу.

### 8.5. Cookie-баннер и аналитика
- Яндекс.Метрика и Google Analytics загружаются сразу, но по умолчанию имеют `analytics_storage: 'denied'`.
- После нажатия «Принять» аналитика активируется.
